data.json - информация о реферальных зависимостях юзеров.

В рамках тестового задания предлагается реализовать микросервис на Django REST framework.


Задание 1:
  - Прочитать и записать в базу данных юзеров и их реферальную зависимость.

  - Назначить каждому юзеру его уровень в реферальной системе.
    V1 - дефолтный уровень
    V2 - всего юзеров в команде 20 человек и  минимум 3 из которых пригласил данный юзер
    V3 - всего юзеров в команде 100 человек и минимум 5 из которых пригласил данный юзер и 3 из них имеют уровень V2
    V4 - всего юзеров в команде 300 человек и минимум 8 из которых пригласил данный юзер и 3 из них имеют уровень V3
    V5 - всего юзеров в команде 800 человек и минимум 12 из которых пригласил данный юзер и 3 из них имеют уровень V4
    V6 - всего юзеров в команде 1500 человек и более 20 из которых пригласил данный юзер и 3 из них имеют уровень V5
  
  - Доп. задание*: Создать endpoint с входящим параметром - id пользователя (из json файла). Результат запроса - информация о пользователе (id, реферальный уровень).

  Структура хранения данных, метод запросов и набор дополнительных данных запроса / ответа определяется разработчиком.


Задание 2*:
  - Рассчитать вознаграждение согласно уровням юзеров и их реферальной зависимости.

  - Каждый юзер делает пополнение на 120$.
    После чего необходимо произвести зачисление вознаграждения юзерам выше его по реферальный системе согласно условиям:
    V1
      - С каждого лично приведенного юзера зарабатывает 30$
    V2
      - С каждого лично приведенного юзера зарабатывает 40$
      - С каждого юзера приведенного юзером V1 его реферальной сетки зарабатывает 10$
    V3
      - С каждого лично приведенного юзера зарабатывает 50$
      - С каждого юзера приведенного юзером V1 его реферальной сетки зарабатывает 20$
      - С каждого юзера приведенного юзером V2 его реферальной сетки зарабатывает 10$
    V4
      - С каждого лично приведенного юзера зарабатывает 60$
      - С каждого юзера приведенного юзером V1 его реферальной сетки зарабатывает 30$
      - С каждого юзера приведенного юзером V2 его реферальной сетки зарабатывает 20$
      - С каждого юзера приведенного юзером V3 его реферальной сетки зарабатывает 10$
    V5
      - С каждого лично приведенного юзера зарабатывает 65$
      - С каждого юзера приведенного юзером V1 его реферальной сетки зарабатывает 35$
      - С каждого юзера приведенного юзером V2 его реферальной сетки зарабатывает 25$
      - С каждого юзера приведенного юзером V3 его реферальной сетки зарабатывает 15$
      - С каждого юзера приведенного юзером V4 его реферальной сетки зарабатывает 5$
    V6
      - С каждого лично приведенного юзера зарабатывает 70$
      - С каждого юзера приведенного юзером V1 его реферальной сетки зарабатывает 40$
      - С каждого юзера приведенного юзером V2 его реферальной сетки зарабатывает 30$
      - С каждого юзера приведенного юзером V3 его реферальной сетки зарабатывает 20$
      - С каждого юзера приведенного юзером V4 его реферальной сетки зарабатывает 10$
      - С каждого юзера приведенного юзером V5 его реферальной сетки зарабатывает 5$

  * Начиная от V3 до V6: если юзер1 лично привел юзера2, а юзер2 достиг такого же уровня как юзер1 или выше, то юзер1 не получает вознаграждение;

  Пример: юзер1(V2) пригласил юзера2(V1), юзер2(V1) пригласил юзера3(V1). Каждый из них пополнил систему на 120$

  юзер1(V2)
    |
    |  юзер1(V2) - получить от юзера2(V1) 40$.
    |  юзер1(V2) - получить от каждого приведенного юзером2(V1) по 10$.
    |  итого вознаграждение юзер1(V2) составит 40$ + 10$
    |  
  юзера2(V1)
    |
    |  юзер2(V1) - получить от юзера3(V1) 30$.
    |  итого вознаграждение юзер2(V1) составит 30$
    |
  юзера3(V1)
    |
    |  юзер3(V1) - не получить вознаграждение тк. у него нету рефералов.
